# Рефлексия

### Введение

В данном уроке мы изучили способ сортировать значения за $O\left(n\right)$ времени! Достигается это за счет вставки элемента в его порядковый индекс в массиве, то есть изначально набор элементов должен определять последовательность итогового отсортированного массива.

На примере чисел:

Дано: `[2, 1, 3, 3, 4, 4, 5, 6]`

Результат: `[1, 2, 3 по 2 раза, 4 по 2 раза, 5, 6]`

То есть мы элемент вставляем по индексу итогового массива, из этого следует, что исходный набор элементов должен содержать ровно последовательность, которая поместится в указанный диапазон.
По-другому называют - Сортировка подсчетом.

Ну, а для того, чтобы сортирвоать к примеру строки, необходимо:
1. Иметь стандартизированный набор, то есть чтобы все строки следовали одному шаблону, в нашем случае: AMN, где A -- это символ из диапазона abcdefgh, а MN -- две цифры.
2. Вычислять хеш строки, тем самым определяя индекс в массиве.

Важно, что если строки не будут по шаблону - возрастает риск коллизий и соответственно нельзя определить не только неповторяющийся индекс, но и корректный индекс, который входит в диапазон итогового массива, который создается заранее определенного размера - который равен максимальному хешу, а если строки будут разные, данное условие перестанет соблюдаться. 

### Хеш-функция

Самым важным в решении подобрать нужную хеш-функцию:

#### Первая итерация

По началу решил написать простую функцию, которая просто суммировала коды значений строки, но данный вариант - не учитывал порядок строк, получался одинаковый кеш, как для `a01`, так `a10`, соответственно коллизии, а также количество уникальных значений было намного меньше -> 7 + 9 + 9 = 25, а для уменьшения вероятности коллизии нужно увеличить количество уникальный значений, а также исключить по минимуму возможность коллизий.

#### Вторая итерация

Тогда в голову пришла вот такая формула:

$diff\left(ord\left(A\right),ord\left(a\right)\right)\cdot100+int\left(M\right)\cdot10+int\left(N\right)$

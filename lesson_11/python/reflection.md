# Рефлексия

## Введение

В данном занятии мы будем изучать алгоритм бинарного поиска, который способен искать элемент в отсортированно массиве за $O\left(\log\left(n\right)\right)$ времени!

Достигается это за счет деления исходного массива на половины до тех пор, пока элемент не будет найден, либо пока массив не иссякнет (указатели не встретятся).

## Алгоритм

Алгоритм прост:
1. Создаются левый и правый указатель (`left, right`), которые указывают на начальный и конечный индекс массива
2. Далее вычисляется средний элемент между указателями ->
# $middle=\frac{\left(left+riht\right)}{2}$

3. сравнивается полученный элемент с целевым
4. если нашли -> заканчиваем поиск
5. если не нашли, сравниваем
	1. если элемент больше целевого, тогда мы сдвигаем правый указатель влево (`right = middle - 1`)
	2. если элемент меньше целевого, тогда мы сдвигаем левый указатель вправо (`left = middle + 1`)
6. алгоритм продолжается до тех пор, пока либо элемент не будет найден, либо пока указатели не встретятся

Таким образом мы все время рассматриваем средний элемент половины массива, после мы делим еще на две половины и рассматриваем только одну из них.
